# Process Goals

## Concept

**Outcome goals** focus on WHAT you want to achieve:
- "Сдать экзамен на отлично"
- "Устроиться в частную клинику"
- "Набрать 50к подписчиков"

**Process goals** focus on HOW you do it:
- "Законспектировать 2 урока в день"
- "Anki-повторение каждый день без пропусков"
- "Написать 1 пост в день"

## Why Process Goals Work Better

Based on research (meta-analysis of 27 experiments in sports):

1. **Control** — You control actions, not outcomes. "Сдать экзамен" creates anxiety; "2 урока в день" is actionable.
2. **Confidence** — Process goals turn challenge into achievable steps, boosting self-efficacy.
3. **Focus** — Daily clarity on what to do, not vague aspiration.

## How to Transform (User's Goals)

| Outcome Goal | Process Goal |
|--------------|--------------|
| "Сдать экзамен на отлично" | "Законспектировать 2 урока в день" |
| "Закрыть отставание" | "1 урок (полный цикл) ежедневно до 7 февраля" |
| "Anki без пропусков" | "Anki-повторение каждый день, первый блок после уроков" |
| "Посты каждый день" | "2 часа на посты каждое утро после завтрака" |
| "Жим 100 кг" | "Зал 2 раза в неделю (среда + суббота)" |
| "Найти партнёра" | Пока отложено — фокус на учёбе |

## Patterns for Process Goals

### Time Blocks
Regular dedicated time for deep work:
- "2 часа на уроки каждый день 12:30–14:45"
- "2.5 часа Anki после уроков"
- "2 часа на посты утром 10:00–12:00"

### Frequency Commitments
Consistent action cadence:
- "1 урок (полный цикл) в день"
- "Anki каждый день без пропусков"
- "1 пост в день"

### Checkpoints
Review milestones:
- "К 9 февраля — отставание закрыто"
- "К 1 марта — все старые уроки в Anki"
- "К 25 марта — готов к экзамену"

### Streaks
Daily habits:
- "23 дня Anki подряд до конца февраля"
- "Посты каждый день февраля"

## When Processing Voice Entries

If user says outcome goal, consider transforming:

```
User: "Нужно подготовиться к экзамену"

BAD: Create task "Подготовиться к экзамену" p1
→ Vague, creates anxiety, unclear next action

GOOD: Create task "Законспектировать урок 46" p2 due:today
→ Specific, actionable, controllable
```

## For Weekly Planning

When generating Process Commitments for ONE Big Thing:

1. **Ask:** What daily/weekly habits would make this inevitable?
2. **Limit:** 3-5 process commitments max
3. **Prioritize:** Time blocks > Frequency > Checkpoints
4. **Create:** Recurring tasks in Todoist with appropriate patterns

## Recurring Task Patterns

Map process goals to Todoist dueString:

| Process | dueString |
|---------|-----------|
| "каждое утро" | "every day at 10am" |
| "каждый день" | "every day" |
| "по будням" | "every weekday" |
| "2 раза в неделю (ср + сб)" | "every wednesday, saturday" |
| "раз в неделю" | "every week" |
| "до пятницы" | "friday" (one-time) |

## Anti-Patterns

Avoid creating process goals that are:
- Too vague: "Учиться" (how much? when?)
- Too ambitious: "4 урока в день" (unsustainable)
- Untrackable: "Думать про экзамен" (not measurable)
- Without time: "Anki когда-нибудь" (no commitment)

Good process goals are:
- Specific in action
- Bounded in time/frequency
- Trackable (done/not done)
- Realistic (учитывают реальное время: 1 урок = 2–2.5 ч)
